기능설명
로그인, 주소록, 마이페이지, 공지사항, 결재관리, 시설관리, 강의목록, 근태목록

  • 로그인
    • 로그인 - Spring Security의 UsernamePasswordAuthenticationFilter가 /loginProc으로 들어온 요청을 가로채 처리
      - 입력한 아이디와 비밀번호가 UserDetailsService와 PasswordEncoder를 통해 검증
      - 검증이 성공하면 로그인한 사용자 정보를 SecurityContextHolder에 저장하고, .defaultSuccessUrl("/main", true)로 리다이렉트
    • 비밀번호 찾기 - Mail Sender API를 이용해서 application-mail.yml 파일에 이미 정의되어있는 이메일로 해당 사원의 임시 비밀번호를 안내문과 함께 발송
      - 이메일이 전송되면, UPDATE 쿼리문을 통해서 bCryptPasswordEncoder를 이용해서 암호화시킨 임시 비밀번호를 데이터베이스를 수정
      - 1을 반환함으로써 임시 비밀번호 발급이 성공
  • 주소록
    • CRUD를 활용해서 데이터를 조회, 수정,삭제 기능을 구현
  • 마이페이지
    • 좌측에는 사원의 개인정보를 확인할 수 있고, 우측에는 FullCalendar API를 사용하여 해당 사원의 일정정보(회의, 강의, 휴가 등)를 확인
    • 마이페이지에는 비밀번호변경, 개인정보수정 버튼이 출력
  • 공지사항
    • 게시판선택, 게시물제목입력, 게시물내용입력, 게시물고정, 파일첨부 가능
    • Quill Editor API를 통해서 게시물내용을 다양한 글꼴모양, 다양한 글꼴크기, 다양한 글꼴색상을 활용
    • 공지사항 게시판은 고정적으로 존재하며, 게시판 카테고리를 새로 개설하거나 삭제 가능
  • 결재목록
    • 근태결재 - 연차, 반차, 병가, 공가를 신청할 때 필요한 결재 신청
      - 결재신청할 때는 신청날짜선택, 신청내용입력, 파일첨부 및 결재선설정
      - 결재선설정은 1명부터 3명까지 자유롭게 설정할 수 있으며, 위쪽에 배치되어있는 사원부터 결재가 차례대로 진행
      - 연차 신청 시 잔여 연차를 초과하면 결재가 진행되지 않습니다
    • 강의결재 - 강사(운영팀)일 경우에만 강의결재버튼 출력
      - 강의를 개설하고자 할 때 필요한 결재를 신청
      - 결재신청할 때는 신청날짜선택, 신청시간선택, 강의실선택, 강의내용입력, 신청내용입력, 파일첨부 및 결재선설정
      - 결재선설정은 1명부터 3명까지 자유롭게 설정할 수 있으며, 위쪽에 배치되어있는 사원부터 결재가 차례대로 진행
      - 이미 등록되어있는 강의시간이나 신청이 진행중인 강의시간은 선택할 수 없으며, 잘못된 강의시간이 입력되지 않도록 다양한 유효성기능을 제공
  • 회의실관리
    • meetingroom테이블에서 회의실 번호, 회의실명, 수용인원, 담당자를 출력 후 담당자와 employee테이블을 join하여 이름, 프로필사진, 사원번호 출력
    • Restful API를 활용해서 Ajax로 데이터를 받아서 뷰로 출력
    • 행정팀일 경우 회의실 등록 및 수정, 삭제버튼 출력
  • 강의목록
    • 행정팀 권한을 가진 사원만 결재된 강의를 수정
    • 강사, 강의실, 강의제목, 강의내용, 강의시간 등 강의에 대한 모든 정보를 수정 가능
    • 강의시간은 이미 등록되어있는 강의시간이나 신청이 진행중인 강의시간은 선택할 수 없지만, 수정 중인 강의의 강의시간은 선택 가능
    • 잘못된 강의시간이 입력되지 않도록 다양한 유효성기능을 제공
  • 채팅
    • 실시간 채팅 기능을 통해 사원들과 1:1 의사소통 가능
    • 읽지 않은 메시지는 사원 이름 옆에 알림 수가 표시
  • 근태목록
    • 근태관리 - 월별로 로그인된 사원의 근태정보(정상출근, 결근 등)를 확인
      - 스케줄러를 사용하여 매일 00시에 모든 사원의 근태 데이터가 자동 생성되며, 데이터 생성 당일에 근태신청서가 있는 사원은 근태신청서에 적힌 근태유형(연차, 병가, 공가 등)으로 변경
      - 매일 23:59에 출퇴근 기록과 근태 유형이 없는 사원은 결석으로 처리되며, 주말에는 휴일로 처리
    • 연차관리 - 월별로 승인된 연차를 확인
      - 강의를 개설하고자 할 때 필요한 결재를 신청

  • 결재목록
    • 근태결재 - 연차, 반차, 병가, 공가를 신청할 때 필요한 결재 신청
      - 결재신청할 때는 신청날짜선택, 신청내용입력, 파일첨부 및 결재선설정
      - 결재선설정은 1명부터 3명까지 자유롭게 설정할 수 있으며, 위쪽에 배치되어있는 사원부터 결재가 차례대로 진행
      - 연차 신청 시 잔여 연차를 초과하면 결재가 진행되지 않습니다
    • 강의결재 - 강사(운영팀)일 경우에만 강의결재버튼 출력
      - 강의를 개설하고자 할 때 필요한 결재를 신청
      - 결재신청할 때는 신청날짜선택, 신청시간선택, 강의실선택, 강의내용입력, 신청내용입력, 파일첨부 및 결재선설정
      - 결재선설정은 1명부터 3명까지 자유롭게 설정할 수 있으며, 위쪽에 배치되어있는 사원부터 결재가 차례대로 진행
      - 이미 등록되어있는 강의시간이나 신청이 진행중인 강의시간은 선택할 수 없으며, 잘못된 강의시간이 입력되지 않도록 다양한 유효성기능을 제공
  • 회의실관리
    • meetingroom테이블에서 회의실 번호, 회의실명, 수용인원, 담당자를 출력 후 담당자와 employee테이블을 join하여 이름, 프로필사진, 사원번호 출력
    • Restful API를 활용해서 Ajax로 데이터를 받아서 뷰로 출력
    • 행정팀일 경우 회의실 등록 및 수정, 삭제버튼 출력
  • 강의목록
    • 행정팀 권한을 가진 사원만 결재된 강의를 수정
    • 강사, 강의실, 강의제목, 강의내용, 강의시간 등 강의에 대한 모든 정보를 수정 가능
    • 강의시간은 이미 등록되어있는 강의시간이나 신청이 진행중인 강의시간은 선택할 수 없지만, 수정 중인 강의의 강의시간은 선택 가능
    • 잘못된 강의시간이 입력되지 않도록 다양한 유효성기능을 제공
  • 채팅
    • 실시간 채팅 기능을 통해 사원들과 1:1 의사소통 가능
    • 읽지 않은 메시지는 사원 이름 옆에 알림 수가 표시
  • 근태목록
    • 근태관리 - 월별로 로그인된 사원의 근태정보(정상출근, 결근 등)를 확인
      - 스케줄러를 사용하여 매일 00시에 모든 사원의 근태 데이터가 자동 생성되며, 데이터 생성 당일에 근태신청서가 있는 사원은 근태신청서에 적힌 근태유형(연차, 병가, 공가 등)으로 변경
      - 매일 23:59에 출퇴근 기록과 근태 유형이 없는 사원은 결석으로 처리되며, 주말에는 휴일로 처리
    • 연차관리 - 월별로 승인된 연차를 확인
      - 강의를 개설하고자 할 때 필요한 결재를 신청
